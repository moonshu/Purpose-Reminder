# PR-AG-013 Plan — 핵심 E2E 시나리오 테스트 스크립트화

## 1. 목표
- MVP 핵심 사용자 플로우를 재사용 가능한 E2E 체크리스트/스크립트로 문서화한다.

## 2. 선행 조건
- PR-AG-011, PR-AG-012 완료

## 3. 변경 대상
- `docs/` 하위 E2E 문서 (예: `docs/e2e-mvp-checklist.md`)
- 필요 시 `Tests/Integration` 보조 테스트 코드

## 4. 구현 단계
1. 핵심 시나리오 5개 정의 (온보딩/정책/목표선택/리마인드/기록)
2. 각 시나리오별 사전조건/입력/기대결과 작성
3. Happy Path와 Edge Case 분리
4. 실패 시 수집해야 할 증적(로그/스크린샷) 기준 정의

## 5. 검증
- 문서: 누락 케이스 없는지 리뷰
- 수동: 실기기에서 핵심 1회차 실행

## 6. 완료 기준 (DoD)
- 시나리오별 시작조건/완료조건/실패기준이 모두 문서화
- QA 담당자가 문서만으로 재수행 가능
- 체크리스트 항목 번호로 실행 결과 기록 가능

## 7. BLOCKED_MANUAL 조건
- 실기기 접근 불가
- 권한 승인 단계에서 반복 실패

## 8. 산출물
- E2E 체크리스트 문서
- 시나리오 결과 기록 템플릿
