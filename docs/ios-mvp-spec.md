# Purpose Reminder iOS MVP Spec

## 1. 문서 목적
- 이 문서는 `Purpose Reminder`의 iOS MVP 기준 문서다.
- 기획/디자인/개발/QA가 같은 기준으로 의사결정하기 위한 단일 기준점이다.
- 기준 우선순위는 다음 순서다.
  1. iOS 정책/실행 가능성
  2. 사용자 마찰 최소화
  3. 빠른 MVP 검증

## 2. 이 앱을 한 문장으로
- `딴짓하려고 앱을 열기 전에, 왜 여는지(목표)를 먼저 정하게 도와주는 앱`

## 3. 30초 요약 (비전공자용)
1. 사용자가 자주 새는 앱(예: Instagram)을 앱 안에서 미리 등록한다.
2. 그 앱을 열려고 하면 iOS의 `Shield` 화면이 먼저 뜬다.
3. 사용자는 `빠른 목표`를 탭하거나 새 목표를 입력한다.
4. 목표가 정해지면 정해진 시간만큼 해당 앱을 사용한다.
5. 중간에 리마인드를 받고, 끝나면 완료/연장/중단을 고른다.
6. 결과는 기록 화면에 쌓여 습관 변화를 확인할 수 있다.

## 4. 한눈에 보는 워크플로
`앱 등록 -> 목표 선택 -> 제한 시간 사용 -> 리마인드 -> 종료 선택 -> 기록`

### 예시 흐름 (Instagram 20분)
1. 사용자가 Instagram을 열려고 한다.
2. iOS Shield에서 `지금 목표를 선택하세요` 화면이 뜬다.
3. 추천 목표 목록에서 `DM 확인만 하고 종료`를 탭한다.
4. Instagram이 20분 동안 임시 허용된다.
5. 15분 시점에 `5분 남았어요` 리마인드가 온다.
6. 20분 종료 시 `완료`, `10분 연장`, `중단` 중 하나를 선택한다.
7. 선택 결과가 오늘 기록에 저장된다.

## 5. 제품 정의
- 앱 성격: 관제탑 앱(설정, 템플릿 관리, 기록 확인)
- 개입 시점: 사용자가 등록된 대상 앱을 열려고 할 때
- 핵심 목표: "무의식적 진입"을 "의식적 목표 기반 사용"으로 바꾸는 것

## 6. iOS 제약 및 구현 원칙
- iOS에서는 Android식 자유 오버레이(타 앱 위 플로팅)가 불가하다.
- 일반 앱이 모든 앱 실행을 임의로 실시간 가로채는 방식도 불가하다.
- 반드시 `Screen Time` 계열 API를 사용한다.
  - `FamilyControls`
  - `ManagedSettings`
  - `DeviceActivity`
- 대상 앱 진입 개입은 `Shield` 기반으로 설계한다.

## 7. MVP 범위 (필수)
1. 대상 앱 선택/관리
2. 앱별 규칙 설정
- 기본 허용 시간 (예: 10/20/30분)
- 리마인드 시점 (예: 종료 5분 전)
3. 목표 설정 방식
- 빠른 시작(최근/즐겨찾기/기본 목표) 우선
- 필요 시 새 목표 직접 입력
4. 세션 동작
- 세션 시작
- 사용 중 리마인드
- 시간 종료 후 완료/연장/중단 선택
5. 기록 화면
- 오늘 세션 수
- 완료율
- 연장/중단 수
6. 단축어(Shortcuts) 연동
- 빠른 목표 시작
- 앱별 즐겨찾기 목표 원탭 시작

## 8. 사용자 플로우 (단계별 상세)
### 8.1 첫 사용 (온보딩)
1. 사용자가 앱을 설치하고 연다.
2. 앱이 "이 기능은 iOS Screen Time 권한이 필요"하다고 설명한다.
3. 사용자가 Screen Time 권한을 허용한다.
4. 리마인드 알림 권한을 허용한다.

완료 기준
- 권한 2종(Screen Time, 알림)이 모두 허용된 상태

### 8.2 초기 설정
1. 사용자가 개입할 대상 앱을 고른다. (예: Instagram, YouTube)
2. 앱별 기본 사용 시간을 정한다. (예: Instagram 20분)
3. 리마인드 시점을 정한다. (예: 종료 5분 전)
4. 기본 목표 문구를 정한다. (예: "필요한 DM만 확인")

완료 기준
- 최소 1개 앱 정책이 저장된 상태

### 8.3 실제 사용 시
1. 사용자가 대상 앱을 열려고 한다.
2. Shield 화면에서 목표 선택을 요구한다.
3. 사용자는 다음 중 하나를 선택한다.
- 빠른 시작(최근/즐겨찾기/기본 목표)
- 새 목표 입력
4. 목표가 확정되면 세션이 시작되고 앱이 임시 허용된다.
5. 리마인드 시점에 알림이 발송된다.
6. 시간이 끝나면 완료/연장/중단 중 하나를 선택한다.
7. 결과가 기록에 저장된다.

## 9. 빠른 목표 설계 (마찰 최소화 핵심)
### 9.1 목표 템플릿 소스
- 즐겨찾기 목표
- 동일 앱 최근 목표
- 전체 최근 목표
- 앱별 기본 목표

### 9.2 추천 정렬 우선순위
1. 즐겨찾기
2. 동일 앱 최근 사용
3. 전체 최근 사용
4. 새 목표 입력

### 9.3 UX 원칙
- 빠른 목표는 탭 즉시 세션 시작 (입력 폼 생략)
- 새 목표 입력은 항상 하단에 보조 옵션으로 노출
- 목표 문구는 짧고 행동형 문장 권장
  - 예: `DM 3개만 답장`, `게시물 1개 업로드 후 종료`

## 10. 기록 화면에서 보여줄 것
- 오늘 세션 수
- 완료율
- 연장 횟수
- 중단 횟수
- 최근 세션 목록(앱, 목표, 시작/종료, 결과)

## 11. 데이터 모델 (MVP 최소)
### 11.1 AppPolicy
- `id`
- `appToken` (FamilyControls 앱 식별값)
- `isActive`
- `defaultDurationMinutes`
- `reminderOffsetMinutes`
- `defaultTemplateId` (nullable)

### 11.2 GoalTemplate
- `id`
- `targetAppToken` (nullable, 공용 템플릿 허용)
- `text`
- `isFavorite`
- `useCount`
- `lastUsedAt`
- `createdAt`

### 11.3 GoalSession
- `id`
- `targetAppToken`
- `templateId` (nullable)
- `goalTextSnapshot`
- `startedAt`
- `endedAt` (nullable)
- `status` (`active`, `completed`, `extended`, `abandoned`, `timed_out`)
- `plannedDurationMinutes`

### 11.4 ReminderEvent
- `id`
- `sessionId`
- `scheduledAt`
- `firedAt` (nullable)
- `action` (`ignored`, `opened`, `completed`, `extended`)

## 12. 상태 머신
- 기본 흐름: `idle -> pending_goal -> active -> reminded -> completed`
- 연장 흐름: `active -> extended`
- 중단 흐름: `active/reminded -> abandoned`
- 시간 초과 흐름: `active/reminded -> timed_out`

## 13. iOS 앱 구성
1. Main App (설정/기록/템플릿/단축어)
2. `ShieldConfigurationExtension`
3. `ShieldActionExtension`
4. `DeviceActivityMonitorExtension`

## 14. 화면 스펙 (MVP)
1. 온보딩/권한 화면
- Screen Time, 알림 권한 안내 및 허용
2. 대상 앱/규칙 설정 화면
- 앱 선택, 기본 시간, 리마인드 설정
3. 목표 템플릿 관리 화면
- 최근, 즐겨찾기, 기본 목표 지정
4. 세션 시작(Shield 기반) 화면
- 빠른 시작 목록 + 새 목표 입력
5. 기록 화면
- 오늘 요약 + 최근 세션

## 15. 단축어(App Intents) 스펙
1. `빠른 목표 시작`
- 파라미터: 대상 앱, 목표 템플릿, 시간
- 동작: 세션 생성 + 임시 허용 시작
2. `즐겨찾기 목표 시작`
- 파라미터 없음 (기본 즐겨찾기/기본 정책 사용)
- 홈 화면/액션 버튼에서 즉시 실행

## 16. 기술 스택
- 언어/UI: Swift, SwiftUI
- 로컬 저장: SwiftData
- 알림: `UNUserNotificationCenter`
- 동기화/서버: MVP 1차 제외 (로컬 우선)

## 17. 일정 (3주 MVP)
1. 1주차
- 권한 온보딩
- 대상 앱 선택/정책 저장
- Shield 개입 골격
2. 2주차
- 빠른 목표(최근/즐겨찾기/기본)
- 세션 시작/종료/연장
- 리마인드 알림
3. 3주차
- 단축어 연동
- 기록 화면
- 안정화 및 QA

## 18. KPI (MVP 검증 지표)
1. 개입 성공률 (대상 앱 진입 시 Shield 개입 비율)
2. 목표 선택률 (빠른 시작 포함)
3. 세션 완료율
4. 연장 비율
5. 재방문율 (D1/D7)

## 19. 리스크 및 대응
1. Screen Time 권한/배포 제약
- 대응: 초기 주차에 권한/엔타이틀먼트 검증 최우선
2. iOS 정책상 오버레이 불가
- 대응: Shield + 알림 UX로 제품 정의 고정
3. 입력 피로
- 대응: 빠른 목표/즐겨찾기/단축어를 MVP 필수 유지

## 20. 자주 헷갈리는 포인트 (FAQ)
1. "이 앱이 다른 앱을 완전히 차단하나요?"
- 완전 차단 앱이 아니라, 목표 기반 사용을 유도하는 개입 앱이다.
2. "왜 매번 목표를 고르게 하나요?"
- 무의식 진입을 줄이는 핵심 장치이기 때문이다.
3. "매번 입력하면 번거롭지 않나요?"
- 그래서 빠른 목표(원탭 시작)를 기본 UX로 둔다.
4. "Android처럼 화면 위 팝업은 안 되나요?"
- iOS 정책상 불가하므로 Shield 중심으로 구현한다.

## 21. 비범위 (MVP 제외)
- Android 동시 개발
- 고급 추천/AI 코칭
- 다중 디바이스 클라우드 동기화
- 소셜/랭킹 기능

## 22. 에이전트 작업 지침 (컨텍스트 유지용)
- 새 작업 세션 시작 시 이 문서를 먼저 읽고 판단한다.
- iOS 정책과 충돌하는 요구가 나오면 `6. iOS 제약 및 구현 원칙` 기준으로 대안을 제시한다.
- 목표 입력 UX는 `빠른 시작 우선` 원칙을 유지한다.
