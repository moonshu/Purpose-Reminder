# Purpose Reminder MVP E2E Checklist

## 1. 실행 목적
- MVP 핵심 플로우를 동일한 절차로 반복 검증한다.
- 실패 시 원인 파악에 필요한 증적을 같은 형식으로 남긴다.

## 2. 공통 실행 조건
- 실기기 필수 시나리오: E2E-01, E2E-02, E2E-03, E2E-04, E2E-06
- 시뮬레이터 가능 시나리오: E2E-05(더미 데이터 가능)
- 사전 문서 확인:
  - `docs/ios-manual-setup-checklist.md`
  - `docs/agent-ticket-spec.md` §8 BLOCKED_MANUAL 포맷

## 3. 시나리오

### E2E-01 온보딩 권한 획득
- 사전 조건: 앱 최초 설치, Screen Time/알림 권한 미허용
- 실행 환경: 실기기

| 단계 | 액션 | 기대 결과 |
|---|---|---|
| 01-1 | 앱 실행 | 온보딩 화면 노출, "계속" 비활성 |
| 01-2 | Screen Time 권한 요청 | 시스템 권한 창 노출 |
| 01-3 | 알림 권한 요청 | 시스템 권한 창 노출 |
| 01-4 | 두 권한 허용 후 계속 탭 | 메인 탭 화면 진입 |

- 실패 기준: 권한 허용 후에도 온보딩 화면에 머물면 실패
- 실패 시 BM 코드: `BM-013-01`

### E2E-02 정책 저장 및 Shield 반영
- 사전 조건: E2E-01 완료
- 실행 환경: 실기기

| 단계 | 액션 | 기대 결과 |
|---|---|---|
| 02-1 | 정책 탭 진입 | 대상 앱 선택 UI 접근 가능 |
| 02-2 | 대상 앱 1개 이상 선택 | 선택 수/정책 draft 생성 |
| 02-3 | 기본 시간/리마인드 조정 후 저장 | 저장 성공 메시지 표시 |
| 02-4 | 화면 재진입 | 저장한 정책 값 유지 |

- 실패 기준: 재진입 시 정책이 사라지거나 저장 실패 반복
- 실패 시 BM 코드: `BM-013-02`

### E2E-03 Shield 개입 후 세션 시작
- 사전 조건: E2E-02 완료, 대상 앱 정책 활성
- 실행 환경: 실기기

| 단계 | 액션 | 기대 결과 |
|---|---|---|
| 03-1 | 대상 앱 실행 시도 | Shield 화면 노출 |
| 03-2 | "목표 선택" 탭 | 앱으로 복귀 후 세션 시작 화면 진입 |
| 03-3 | 추천 목표 또는 직접 입력으로 시작 | GoalSession 생성(status: active) |
| 03-4 | 세션 시작 섹션 확인 | 목표 텍스트/계획 시간 표시 |

- 실패 기준: 목표 시작 후 active 세션 생성 실패
- 실패 시 BM 코드: `BM-013-01`

### E2E-04 리마인드 수신 및 세션 종료
- 사전 조건: E2E-03 완료, 알림 권한 허용
- 실행 환경: 실기기

| 단계 | 액션 | 기대 결과 |
|---|---|---|
| 04-1 | 세션 시작 후 대기 | 종료 N분 전 리마인드 알림 수신 |
| 04-2 | 세션 종료 액션 선택(완료/연장/중단) | 상태 변경 및 세션 저장 |
| 04-3 | 동일 세션 리마인드 이벤트 확인 | ReminderEvent fired/action 기록 |

- 실패 기준: 리마인드 미수신 또는 종료 상태 저장 실패
- 실패 시 BM 코드: `BM-010-01`

### E2E-05 기록 화면 검증
- 사전 조건: 종료된 세션 1개 이상
- 실행 환경: 시뮬레이터 또는 실기기

| 단계 | 액션 | 기대 결과 |
|---|---|---|
| 05-1 | 기록 탭 진입 | 오늘 요약 섹션 노출 |
| 05-2 | 최근 7일 목록 확인 | 최신 시작 시각 기준 내림차순 |
| 05-3 | 상태/목표 텍스트 확인 | 각 row의 상태 라벨과 목표 표시 |

- 실패 기준: 종료된 세션이 요약/목록에 반영되지 않음

### E2E-06 App Intents 실행
- 사전 조건: 활성 정책 1개 이상, (Favorite는 즐겨찾기 템플릿 1개 이상)
- 실행 환경: 실기기

| 단계 | 액션 | 기대 결과 |
|---|---|---|
| 06-1 | Shortcuts 앱에서 Purpose Reminder 확인 | `빠른 목표 시작`, `즐겨찾기 목표 시작` 노출 |
| 06-2 | 빠른 목표 시작 실행 | 세션 시작 dialog 표시 |
| 06-3 | 즐겨찾기 목표 시작 실행 | 세션 시작 dialog 표시 |

- 실패 기준: Intent 미노출 또는 실행 시 세션 미생성
- 실패 시 BM 코드: `BM-012-01`, `BM-012-02`

## 4. Edge Case 체크
- EE-01: Screen Time 권한 거부 후 계속 버튼 비활성 유지
- EE-02: 알림 권한 거부 시 세션은 시작되지만 리마인드 경고 노출
- EE-03: 활성 정책 없는 상태에서 세션 시작 시 안내 메시지 표시
- EE-04: 진행 중 세션 상태에서 추가 시작 시 차단 메시지 표시
- EE-05: 즐겨찾기 없는 상태에서 Favorite Intent 실행 시 안내 메시지 표시
